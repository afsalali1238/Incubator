# 90-Day Calendar — Airbnb
Time budget: Full-time (3 founders)
Cadence: Weekly sprints
Board meeting: Every 2 weeks

---

## Week 1–2 | Idea → Proof of Concept

**Theme: Validate the riskiest assumption**

- [x] CEO interview — thesis locked, riskiest assumption named
- [x] Autonomous market research (20-search cap)
- [x] 9-panel findings report delivered
- [x] Founding team roster designed (5 agents, sequenced)
- [x] Community & Supply Lead hired
- [x] Trust & Safety Designer hired
- [x] INCUBATOR.md written to workspace root
- [ ] Identify IDSA Design Conference as anchor event (Aug 2008, SF, hotels sold out)
- [ ] Map 10 candidate hosts via Craigslist + SF design community

**Exit criterion:** 10 candidate hosts identified, 3 confirmed willing to be photographed

---

## Week 3–4 | Proof of Concept → First Bookings

**Theme: Manual end-to-end run**

- [ ] Visit and photograph all 3 hosts (3 rooms each, natural light)
- [ ] Write listing copy for each host (neighborhood description, host bio)
- [ ] Price all listings at hotel rate −35% (anchor: $80–90/night vs $125+ hotels)
- [ ] Build v1 site: static page, PayPal escrow manually managed, email confirmation
- [ ] Design and launch mutual-review flow (post-checkout email, 1–5 stars + free text)
- [ ] Go live 3 weeks before IDSA conference
- [ ] Board meeting #1 — review supply quality, pricing, trust flow

**Exit criterion:** 3 listings live, first booking received before conference date

---

## Week 5–6 | First Bookings → Validation

**Theme: IDSA conference test**

- [ ] All 3 listings booked for IDSA conference dates
- [ ] CEO personally reachable for every host's first check-in
- [ ] Guests check out — collect NPS within 24h
- [ ] Host NPS collected within 48h of checkout
- [ ] Board meeting #2 — health score every agent, validate/pivot/kill decision

**Validation gate:**
- ✅ BUILD: 2+ of 3 hosts willing to repeat + 1+ guest books a second stay
- 🟡 REFINE: Bookings completed but host or guest friction — fix the friction, retest
- 🔴 PIVOT: Hosts refuse to repeat OR guests report safety/quality concern — rethink trust layer

---

## Week 7–8 | Validation → Self-Serve MVP

**Theme: Remove the founder from the loop**

- [ ] Hire Product Engineer (post-validation)
- [ ] Build self-serve listing creation (host fills form, no CEO visit required)
- [ ] Build self-serve booking flow (guest books, PayPal processes, host confirms)
- [ ] Expand to 10 SF listings beyond IDSA hosts (referral from first 3)
- [ ] DNC (Democratic National Convention, Denver) — identify as next anchor event
- [ ] Board meeting #3 — product health, supply velocity, first self-serve booking

**Exit criterion:** First booking completed with zero founder involvement

---

## Week 9–10 | MVP → Multi-Event

**Theme: Prove repeatability**

- [ ] Run DNC as second event test (Denver)
- [ ] SF: 20+ active listings, self-serve fully live
- [ ] Test referral mechanic — ask top hosts for 3 referrals each
- [ ] First organic (non-founder) listing created
- [ ] PMF survey: "How would you feel if you could no longer use Airbnb?" (very disappointed / somewhat / not disappointed)
- [ ] Board meeting #4 — PMF score, second-city supply health

**Exit criterion:** >30% "very disappointed" in PMF survey

---

## Week 11–12 | Scale Signal → Fundraise Prep

**Theme: PMF confirmed → grow or raise**

- [ ] If PMF >40% "very disappointed" → hire Growth & Demand Lead
- [ ] If VC track → deliver Phase 6 pitch deck (10 slides)
- [ ] YC W2009 application (deadline ~Oct 2008)
- [ ] Expand to 3rd city or 3rd major event
- [ ] First press hit (TechCrunch or local SF tech media)
- [ ] Board meeting #5 — full roster review, 90-day debrief

**Exit criterion (90-day end):** PMF confirmed, 3 cities, self-serve end-to-end, fundraise path clear
