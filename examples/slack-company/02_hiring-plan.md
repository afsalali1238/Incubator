# Hiring Plan — Slack

## The Org Chart

```
CEO (The Founder)
├── Product/UX Designer (MVP / Client)
├── Backend / Architecture Engineer (MVP / Server)
└── Growth / Onboarding Specialist (Launch / GTM)
```

## The Hire Sequence

1. **Product/UX Designer** — Hired first because the only way we beat email is if the app is deeply joyful and intuitive to use.
2. **Backend / Architecture Engineer** — Hired second to untangle the chat server from the dead game's codebase and ensure message search is lightning fast.
3. **Growth / Onboarding Specialist** — Hired third to build the "bottom-up" adoption funnel (making it easy for one designer to invite their 5 team members).

## The Roster

### Product/UX Designer
**De-risks:** User retention and daily active usage.
**First win:** Designing a chat interface that feels less like a corporate tool and more like an informal, persistent living room for a team.
**Skill/Brief:** [team/ux-designer.md](./team/ux-designer.md)
