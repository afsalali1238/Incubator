# 90-Day Build Calendar — Slack

## Weeks 1–2 — Validate (Idea stage)
**Owner:** Product/UX Designer
- Strip the internal chat tool down to its absolute essentials (channels, DMs, search).
- Interview 3 friendly startup teams about their internal email pain points.
**Exit:** 3 external teams agree to try our chat tool instead of email for one week.

## Weeks 3–6 — Build the core product (MVP stage)
**Owner:** Backend Engineer & UX Designer
- Separate the chat backend from the Glitch game servers.
- Build a polished, consumer-grade web client.
- Add basic file uploading and a robust search bar.
**Exit:** The 3 friendly teams successfully transition 80% of their internal comms to the app.

## Weeks 7–10 — Iterate toward evidence (MVP → Launch)
**Owner:** Growth Specialist
- Build a frictionless team-creation and invite flow.
- Add integrations for GitHub and Jira so developers get immediate value.
**Exit:** Organic viral growth begins (teams inviting other teams or contractors).

## Weeks 11–12 — Launch readiness (Launch stage)
**Owner:** All
- Implement the "freemium" billing model (pay to unlock search history past 10,000 messages).
- Launch publicly via tech press.
**Exit:** 100 teams active daily, with at least 5 converting to paid.
