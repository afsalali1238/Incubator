# 90-Day Build Calendar — 23andMe

## Weeks 1–2 — Validate (Idea stage)
**Owner:** Clinical / Regulatory SME
- Define the legal boundary: what can we tell consumers about their DNA without being classified as a medical diagnostic device?
- Map the FDA's current stance on D2C health information.
**Exit:** A legal memo confirming we can launch the "ancestry" features without immediate shutdown.

## Weeks 3–6 — Build the core product (MVP stage)
**Owner:** Geneticist / Data Scientist
- Secure a contract with an existing CLIA-certified laboratory to process saliva samples.
- Build the mapping table that links specific SNPs (Single Nucleotide Polymorphisms) to known ancestry populations.
**Exit:** The lab successfully processes a test sample and returns a valid raw data file.

## Weeks 7–10 — Iterate toward evidence (MVP → Launch)
**Owner:** Consumer Trust Designer
- Design the physical spit kit (the tube, the return envelope, the instructions).
- Build the web interface to display the results securely.
**Exit:** A 10-person beta test successfully spits in the tubes, mails them, and reads their results on the web app without confusion.

## Weeks 11–12 — Launch readiness (Launch stage)
**Owner:** All
- Finalize the privacy policy (critical: ensure users can opt-in to anonymized research).
- Manufacture the first 1,000 physical kits.
**Exit:** Open public sales and ship the first 100 kits.
