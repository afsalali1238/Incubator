# 90-Day Build Calendar — Oculus VR

## Weeks 1–2 — Validate (Idea stage)
**Owner:** Optical / Sensor Engineer
- Buy off-the-shelf IMU sensors (accelerometer, gyroscope).
- Write custom firmware to poll the sensor at 1000Hz.
**Exit:** Proof that motion data can be retrieved with <10ms of latency on the hardware side.

## Weeks 3–6 — Build the core product (MVP stage)
**Owner:** Hardware Hacker / Prototyper
- Disassemble a 5.6-inch smartphone screen.
- Duct-tape the screen to a pair of ski goggles.
- Integrate the fast IMU sensor and write the driver to warp the PC rendering to counteract lens distortion.
**Exit:** A functional, ugly, duct-taped headset that displays a simple 3D room.

## Weeks 7–10 — Iterate toward evidence (MVP → Launch)
**Owner:** Developer Relations Lead
- Fly to id Software / John Carmack to demo the duct-taped prototype.
- Get a classic game (Doom 3 BFG Edition) ported to support the headset.
**Exit:** A legendary game developer publicly vouches that the latency is low enough to prevent sickness.

## Weeks 11–12 — Launch readiness (Launch stage)
**Owner:** All
- Design a slightly more polished plastic housing (the "DK1").
- Draft the Kickstarter campaign aimed strictly at developers.
**Exit:** Launch the Kickstarter to fund the first manufacturing run of 1,000 units.
