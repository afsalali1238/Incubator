# Examples

Real output from running `project-ceo` on **Terrarium Planet** — a vivarium and terrarium
care companion app. This is what the skill actually produces.

## Files

```
terrarium-planet/
├── 00_charter.md                          ← read this first
├── roster.md                              ← the live team registry
├── team/
│   └── systems-retention-designer.md     ← one agent persona brief
└── skills/
    └── systems-designer/SKILL.md         ← one installable agent skill
```

The full run also produces `01_findings-report.html` (the 9-panel research report) and
`02_hiring-plan.md` / `03_90-day-calendar.md` — those aren't included here to keep the
example readable, but the charter and roster give you the shape of the full output.

## Notes

- The charter is one screen — thesis, verdict, riskiest assumption, scope fence, org, next action.
- The roster tracks agent health and current tasks. The 🔴 engineer entry shows the CEO's
  judgment: don't hire before the validation test proves the habit.
- The Systems Designer skill contains real domain substance — BJ Fogg's Tiny Habits model
  applied specifically to vivarium care, the "first run experience" test, the day-3 dropout
  check, actual dart-frog care parameters. Not role-play flavor.
- The IP Lawyer in the SPAWN section is a one-off temp — trademark check complete, never made
  into a standing hire.
